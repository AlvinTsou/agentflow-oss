agentflow-oss

一個 CLI-first 的 workflow engine，用來讓 AI-assisted coding、review、quality gates、provider routing、sprint state 與 carry-over workflows 可以重複、可追蹤、可維護。

快速開始 查看 Roadmap
$ pnpm ag init sdd --problem "..."
agentflow-engine --sprint-runner

為何選擇 AgentFlow？

傳統的 AI 編程助手大多為黑盒子的聊天介面。AgentFlow 則是直接在您的程式庫中導入結構化的狀態機與自動化的品質關卡。

🔄

品質循環 Rubrics

每次程式產出皆會經歷嚴格的「審查-修復」品質循環，在提交之前會依照定義的標準進行評分驗證。

📍

Git Checkpoints

引擎會自動將每個步驟的執行結果提交至本地 Git 工作區，方便您隨時回滾、檢視變更或無縫續傳。

🛡️

Middleman 安全防護

內建 Provider 路由層。防止模型能力不匹配、記錄每次路由決策，並自動對敏感密鑰進行脫敏。

📋

Readiness 準備度審計

在代碼合併前，自動將剩餘的 Issues 整理成結構化的 Blocked (阻礙)、Deferred (延期) 或 Nit (微調) 訊號。

內建 Recipes 工作流

Recipes 定義了 AI 引擎編排步驟的方式。在下方探索我們核心的工作流。

Recipe: sdd
Mode: real-codebase
Quality Loop: 啟用 (分數目標: 9+)

Spec-Driven Development (sdd) 專為編寫實體代碼而設計。它會分析需求 brief、原型設計變更、執行本地測試、在品質循環中自動修復錯誤，並在最後提交乾淨的實作。

1. 討論需求
→
2. 探索代碼庫
→
3. 程式原型
→
4. 審查評分
→
5. 自動修復循環
→
6. 整理打包
Recipe: research
Mode: synthetic
Concurrency: 並行子代理

一個完整的研究引擎。它會拆解並規劃研究主題、同時派發多個並行子代理進行資料檢索與調查、合成調查結果、透過獨立審查者進行評估批判，最後生成完整研究報告。

1. 定義主題
→
2. 規劃子問題
→
3. 並行調查 (Parallel Agents)
→
4. 合成整理
→
5. 批判審查
→
6. 最終定稿
Recipe: release-readiness
Mode: real-codebase
Audit Type: 版本與文件

在發布新版本前自動執行代碼庫檢查。它會核對 Changelog 是否包含最近的提交、比對 package.json 的版本是否與 Git tag 相符，並確認代碼中的變更已同步至 Markdown 文件。

1. 審計 Changelog
→
2. 檢查 SemVer 版本
→
3. 驗證文件同步
→
4. 導出 readiness.json

Week 8 活躍維護中 進度更新時間：2026-06-22

公開 repository 的 OSS baseline 已經到位：CI、issue templates、security policy、contribution docs、CLI reference、architecture docs 與 active roadmap 都已建立。Core engine 已能執行 sprint recipes、保存 artifacts 與 state、套用 quality loops、支援 resume，並產生 readiness signals。Middleman layer 已支援 provider capability validation、route metadata、security profiles，以及可回放的 route audit output。已完成 recipe set 包含 mini、research、sdd、release-readiness、pr-review、security-review 與 api-design-review。

  • OSS 基建完備

    以 Apache-2.0 協議開源，完成貢獻指南、安全政策、PR 範本與基本 CI 設定。

  • 核心 CLI 工作流

    支援：init, run, resume, status, replay, approve, request-changes, force-pass, 和 resolve。

  • Middleman 整合路由

    內建路由層，具備模型能力驗證、路徑元數據紀錄，以及安全過濾 Profile。

  • Release-Readiness 強化

    已完成邊界情況強化、CLI 終端介面優化，以及自訂遮蔽設定驗證規則。

  • 事件驅動與管理合約

    已實現背景 daemon 觸發器、並行信號量、串流檢查點記錄，以及管理平台資料合約與測試驗證。

功能明細

詳細列出目前已就緒的功能，以及我們正在開發中的下一階段目標。

已發佈功能

  • CLI-first 運作引擎

    專為終端機設計，接管本地工作區與 Git status checkpoint 安全存檔，並流暢協調代理任務。

  • Middleman 路由機制

    依模型能力做 Provider 路由分發，內建安全 Profile 與請求驗證機制。

  • 品質循環 Rubric 驗證

    自動化的產出-審查-修正循環，針對設定之 Rubrics 進行打分與迭代，直到符合標準為止。

  • Recipe Workflows 與 Review Automation

    已提供 release-readiness 與 PR review recipes，並支援 git diff ingestion 作為結構化 review context。

  • Security Review Recipe

    專為身份驗證、數據存取和不安全日誌設計的 3 步驟 recipe，並提供結構化 Verdict。

  • 事件觸發與並行執行 (v1.5)

    支援背景 daemon (cron/fs-watch/git-hook) 觸發事件，並藉由信號量控制並行執行。

  • API 設計審查工作流 (v1.6)

    專為 RESTful 端點、Naming Style 及破壞性變更進行品質管控的工作流。

  • 管理平台 API 與檢查點 (v1.7)

    透過 Node 原生 API 與本地 JSONL 資料庫，結構化輸出階段檢查點、事件及 Sprint 摘要。

未來規劃中
