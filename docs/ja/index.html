AI-assisted coding、review、quality gates、provider routing、sprint state、carry-over workflows を、再現可能で追跡しやすい形にする CLI-first の workflow engine。

なぜ AgentFlow なのか？

従来の AI 支援コーディングはブラックボックスなチャットUIでした。AgentFlow は、開発プロセスに直接、構造化された状態遷移と自動品質ゲート（Quality Gate）を導入します。

🔄

品質ループ (Rubrics)

すべての成果物は厳密な「作成・検証・修正」ループを経由します。コードのコミット前に、設定した判定基準に基づいて自動評価を行います。

📍

Git セーブポイント

エンジンが自動で各ステップの変更をローカル Git ワークスペースにコミットします。いつでも簡単に巻き戻しや、途中からの再開が可能です。

🛡️

Middleman ガード

プロバイダの抽象ルーティング層。LLMの機能ミスマッチを防ぎ、呼び出し履歴を監査ログに記録、機密情報の自動マスクを行います。

📋

Readiness 監査

プロダクションへのマージ前に、コードや課題の状態を Blocked (阻害項目)、Deferred (保留)、Nit (軽微な修正) に分類して整理します。

組み込み Recipes ワークフロー

Recipes は AI ワークフローの各手順を定義します。コアとなる処理の流れを以下で探索できます。

Recipe: sdd
Mode: real-codebase
Quality Loop: 有効 (目標スコア: 9+)

Spec-Driven Development (sdd) はコードの自動開発に特化しています。仕様定義の相談から、モック作成、ローカルテストの実行と自動修正ループ、そして最終成果物の反映までを行います。

1. 仕様相談
→
2. コード探索
→
3. プロトタイプ作成
→
4. 自動レビュー評価
→
5. 修正ループ
→
6. コード整理
Recipe: research
Mode: synthetic
Concurrency: 並列サブエージェント

高度なリサーチエンジン。初期クエリをサブ質問に分解し、複数のサブエージェントを並列に走らせて調査を行い、内容を統合したのち独立したレビューアーが評価・完成させます。

1. テーマ決定
→
2. 質問の分解
→
3. 並列調査 (Parallel Agents)
→
4. 情報の統合
→
5. レビュー評価
→
6. 報告書完成
Recipe: release-readiness
Mode: real-codebase
Audit Type: バージョンと文書

公開前の自動リポジトリ監査。Changelog に最近のコミット漏れがないかの確認、package.json のバージョンと Git タグの整合性検証、およびドキュメントの同期状態をチェックします。

1. Changelog 監査
→
2. バージョン整合性
→
3. ドキュメント同期検証
→
4. readiness.json 作成

プロジェクトの進捗

Week 3 開発中 ステータス更新日：2026-06-10

公開 repository の OSS baseline は整備済みです。CI、issue templates、security policy、contribution docs、CLI reference、architecture docs、active roadmap が用意されています。Core engine は sprint recipes の実行、artifacts と state の保存、quality loops、resume、readiness signals をサポートしています。Middleman layer は provider capability validation、route metadata の記録、provider calls 前の security profiles を提供します。

  • OSS 基盤の整備完了

    Apache-2.0 ライセンスに基づく公開メンテナンス、貢献ガイド、セキュリティポリシー、各種テンプレート、CI 設定が完了しました。

  • コア CLI ワークフロー

    以下のワークフローに対応：init, run, resume, status, replay, approve, request-changes, force-pass, resolve。

  • Middleman 統合ルーティング

    機能検証、ルートメタデータの記録、セキュリティプロファイルを適用する内部ルーティング層をリリースしました。

  • リリース準備機能の強化

    現在、release-readiness テスト、fixture カバレッジ、カスタム秘密情報マスク機能の実装、および v0.2.0 リリースノートの下書き作成に注力しています。

機能のロードマップ

現在リリース済みの機能と、今後の開発予定機能の詳細な内訳です。

実装済みの機能

  • CLI-first 実行エンジン

    ターミナル運用に特化。ローカルワークスペース管理、Git セーブポイント、各エージェントタスクの進行を管理。

  • Middleman ルーティング層

    プロバイダの自動選択、要件別のモデル配分、セキュリティポリシーによる呼び出し制限。

  • 品質ループ Rubric 検証

    成果物コードの自動テスト・修正ループ。設定した Rubric の品質を満たすまで繰り返し調整を実行。

  • 実行ステートと準備度

    全ログ、生成コード、消費トークンの記録。マージ前にプロジェクト全体の準備度シグナルを生成。

今後の予定

  • Release Readiness 監査テスト

    パッケージやドキュメントの Fixture データを活用した、オフラインでの安全なリポジトリ確認テスト。

  • カスタム秘密情報マスク

    正規表現を設定ファイルに記述し、社外に送信されるリクエスト内の重要トークンをマスク処理。

  • 呼び出し履歴のリプレイ出力

    Middleman が行ったルーティングの決定やコールログを、見やすい形式でターミナルに出力する監査機能。

    • Web ダッシュボード UI

    CLI 内部のアーキテクチャが完全に安定した段階で提供を開始する、視覚的コントロールパネル。
