### Security Policy Source Audit

We audited the documented security posture against live source for credential
handling, prompt redaction, management server behavior, webhook delivery, and
local state files.

Confirmed mismatches and fixes:

- Replaced overbroad "secrets never leave the machine" wording with the more
  precise credential boundary: provider credentials are used for outbound
  provider authentication but are not written to sprint artifacts, state files,
  or logs by agentflow-oss.
- Extended Middleman redaction to cover documented Bearer tokens, connection
  strings, and suspicious secret assignments in addition to API-key/private-key
  patterns and configured custom redactions.
- Hardened the local management server to bind to `127.0.0.1` by default,
  restrict CORS to trusted local origins by default, validate supported web
  actions, and append typed web-originated events.
- Updated management API documentation to state the loopback/no-built-in-auth
  boundary explicitly.

## Remaining Watch Items

- The management server intentionally has no built-in authentication. It should
  remain loopback-only unless an external access-control layer is added.
- Webhook delivery is user-configured outbound HTTP. Users should only configure
  trusted destinations because event payloads can include sprint metadata.
