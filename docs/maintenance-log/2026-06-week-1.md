# Week 1 Maintenance Log

This log summarizes the maintenance and development activities performed during the first week of publishing `agentflow-oss`.

## Activities Summary

### Middleman Expansion Milestone

We successfully expanded the Middleman provider routing and safety layer to increase robustness, audibility, and safety.

1. **Provider Capability Registry (`src/middleman/capabilities.ts`)**:
   - Implemented a declarative feature matrix mapping provider feature support (streaming, tool calls, JSON format, smoke tests, limits, and timeouts).

2. **Capability Validation & Route Warnings (`src/middleman/middleman.ts`)**:
   - Added validation before dispatching requests.
   - Throws immediate descriptive errors if a provider lacks required capabilities (e.g. tool-calls).
   - Gracefully handles streaming fallbacks with a route warning when streaming is requested on non-streaming providers.
   - Exposed route decisions inside the returned `StepResult`.

3. **Security Profiles & Route Audit (`src/middleman/policy.ts`)**:
   - Extended policy configuration with named security profiles: `default` (redact secrets), `strict` (block secrets and throw), and `off` (disable scanner).
   - Integrated route decisions (resolved provider, model, reason, warnings, policy profile) directly into the sprint events logged to `events.jsonl` under the `phase` event.

4. **Integration & Offline Testing (`tests/poc-middleman.ts`)**:
   - Added exhaustive offline tests covering security profiles, unsupported capability failure, stream warning preservation, and route metadata verification.

### Concept Documentation

We completed the core workflow model documentation to improve project onboarding and conceptual clarity.

1. **State-Machine Concept Doc (`docs/concepts/state-machine.md`)**:
   - Explained AgentFlow's engine-owned state machine, the iterative Produce-Review-Fix Quality Loop, Clean-Context isolation, and Git-backed checkpoints for crash-resilience and rollbacks.
