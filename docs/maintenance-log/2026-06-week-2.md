# Week 2 Maintenance Log

This log summarizes the maintenance and development activities performed during the second week of publishing `agentflow-oss`.

## Activities Summary

### Release-Readiness Recipe & Fixture Coverage

We designed, integrated, and validated a built-in workflow recipe to automate release sanity checks.

1. **Recipe Definition (`recipes/release-readiness.json`)**:
   - Defined a 3-step workflow recipe: `audit-changelog` (analyzes git logs vs `CHANGELOG.md`), `check-version` (verifies `package.json` version bumps against release tags), and `validate-docs` (ensures new options or CLI flags have documentation coverage).

2. **Integration & Execution Support (`src/recipe/registry.ts`)**:
   - Integrated `release-readiness` into the dynamic recipe registry loader, allowing it to be initialized and run in standard sprints.

3. **Offline Mock & Fixture Coverage (`tests/poc-release-readiness.ts`)**:
   - Created test fixtures under `tests/fixtures/release-readiness/` covering four realistic release-prep states: `missing-changelog`, `version-mismatch`, `missing-docs`, and `release-ready`.
   - Expanded the test suite to mock Middleman LLM calls and verify `runSprint` execution.
   - Asserted that audit report findings are correctly parsed and categorised as `blocking`, `deferred`, or `nit`.

### Custom Secret Redaction Patterns

We introduced support for project-specific redaction rules to prevent leakage of proprietary keys or custom identifiers.

1. **Configurable Redaction Policy (`src/middleman/policy.ts`)**:
   - Extended `MiddlemanPolicy` to accept a `customRedactions` array containing regex patterns.
   - Safely compiles custom patterns on the fly.
   - Integrates custom scanning with existing security profiles: redacting under `default`, blocking and raising `MiddlemanPolicyError` under `strict`, and bypassing under `off`.

2. **Policy Verification (`tests/poc-middleman.ts`)**:
   - Added test assertions to verify custom regex scanning across all security profiles and ensure ordinary code identifiers do not trigger false positives.

### Documentation Updates

1. **Provider Routing Documentation (`docs/provider-routing.md`)**:
   - Documented the new custom secret redaction policy structure and provided configuration examples for `agentflow.config.json`.
