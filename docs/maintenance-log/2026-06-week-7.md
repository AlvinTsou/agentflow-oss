### Conditional Step Execution Implementation (A-3)

We implemented conditional execution at the step level, absorbing key autonomy primitives.

1. **Recipe Types (`src/recipe/types.ts`)**:
   - Added the optional `condition?: (ctx: StepContext) => boolean` field to `StepDef` interface.
2. **Sprint Engine (`src/workflow/sprint-engine.ts`)**:
   - Integrated condition check during step execution. The step is skipped immediately if the condition evaluates to false.
3. **State Store (`src/workflow/state-store.ts`)**:
   - Registered the new `"step-condition-skipped"` event in `SprintEventType`.
4. **Replay CLI Rendering (`ag-replay.ts`)**:
   - Added support for parsing and rendering `step-skipped` and `step-condition-skipped` events. Skipped steps display cleanly as `"skipped"` or `"cond-skipped"` in the Steps status table.
5. **Offline Proof-of-Concept (`tests/poc-conditional-step.ts`)**:
   - Implemented a complete test suite to verify the condition skipping behavior and its integration with `ag-replay` CLI rendering.
