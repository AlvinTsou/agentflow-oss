agentflow-oss

一个 CLI-first 的 workflow engine，用于让 AI-assisted coding、review、quality gates、provider routing、sprint state 和 carry-over workflows 变得可重复、可追踪、可维护。

快速开始 查看 Roadmap
$ pnpm ag init sdd --problem "..."
agentflow-engine --sprint-runner

为何选择 AgentFlow？

传统的 AI 编程助手大多为黑盒子的聊天界面。AgentFlow 则是直接在您的代码库中引入结构化的状态机和自动化的质量关卡。

🔄

质量循环 Rubrics

每次代码产出都会经历严格的“审查-修复”质量循环，在提交之前会按照定义的标准进行评分验证。

📍

Git Checkpoints

引擎会自动将每个步骤的执行结果提交至本地 Git 工作区，方便您随时回滚、检视变更或无缝续传。

🛡️

Middleman 安全防护

内置 Provider 路由层。防止模型能力不匹配、记录每次路由决策，并自动对敏感密钥进行脱敏。

📋

Readiness 准备度审计

在代码合并前，自动将剩余的 Issues 整理成结构化的 Blocked (阻碍)、Deferred (延期) 或 Nit (微调) 信号。

内置 Recipes 工作流

Recipes 定义了 AI 引擎编排步骤的方式。在下方探索我们核心的工作流。

Recipe: sdd
Mode: real-codebase
Quality Loop: 启用 (分数目标: 9+)

Spec-Driven Development (sdd) 专为编写实体代码而设计。它会分析需求 brief、原型设计变更、运行本地测试、在质量循环中自动修复错误，并在最后提交干净的实现。

1. 讨论需求
→
2. 探索代码库
→
3. 代码原型
→
4. 审查评分
→
5. 自动修复循环
→
6. 整理打包
Recipe: research
Mode: synthetic
Concurrency: 并行子代理

一个完整的研究引擎。它会拆解并规划研究主题、同时派发多个并行子代理进行资料检索与调查、合成调查结果、通过独立审查者进行评估批判，最后生成完整研究报告。

1. 定义主题
→
2. 规划子问题
→
3. 并行调查 (Parallel Agents)
→
4. 合成整理
→
5. 批判审查
→
6. 最终定稿
Recipe: release-readiness
Mode: real-codebase
Audit Type: 版本与文档

在发布新版本前自动执行代码库检查。它会核对 Changelog 是否包含最近的提交、比对 package.json 的版本是否与 Git tag 相符，并确认代码中的变更已同步至 Markdown 文件。

1. 审计 Changelog
→
2. 检查 SemVer 版本
→
3. 验证文档同步
→
4. 导出 readiness.json

项目进展

Week 6 活跃维护中 进度更新时间：2026-06-14

公开 repository 的 OSS baseline 已经到位：CI、issue templates、security policy、contribution docs、CLI reference、architecture docs 和 active roadmap 都已建立。Core engine 已能运行 sprint recipes、保存 artifacts 和 state、应用 quality loops、支持 resume，并生成 readiness signals。Middleman layer 已支持 provider capability validation、route metadata、security profiles，以及可回放的 route audit output。已完成 recipe set 包含 mini、research、sdd、release-readiness、pr-review 与 security-review。

  • OSS 基建完备

    以 Apache-2.0 协议开源，完成贡献指南、安全政策、PR 模板与基本 CI 配置。

  • 核心 CLI 工作流

    支持：init, run, resume, status, replay, approve, request-changes, force-pass, 和 resolve。

  • Middleman 整合路由

    内置路由层，具备模型能力验证、路径元数据记录，以及安全过滤 Profile。

  • Release-Readiness 强化

    当前重点是 Pages accuracy、route audit examples，以及下一阶段的扩展功能。

功能明细

详细列出目前已就绪的功能，以及我们正在开发中的下一阶段目标。

已发布功能

  • CLI-first 运行引擎

    专为终端设计，接管本地工作区与 Git status checkpoint 安全存档，并流畅协调代理任务。

  • Middleman 路由机制

    依模型能力做 Provider 路由分流，内置安全 Profile 与请求验证机制。

  • 质量循环 Rubric 验证

    自动化的产出-审查-修正循环，针对设定之 Rubrics 进行评分与迭代，直到符合标准为止。

  • Recipe Workflows 与 Review Automation

    已提供 release-readiness 与 PR review recipes，并支持 git diff ingestion 作为结构化 review context。

  • Security Review Recipe

    专为身份验证、数据访问和不安全日志设计的 3 步骤 recipe，并提供结构化 Verdict。

未来规划中

    持续同步 multilingual Pages、recipe guides、route audit examples 与 maintenance logs。

    • 更多 Sanitized Replay Fixtures

    扩充 public-safe route audit replay 范例，让 maintainer verification 更容易追踪。

    稳定版 Web 管理界面

    待 CLI 引擎的底层架构完全稳定后，提供轻量级的网页 dashboard 控制台。
