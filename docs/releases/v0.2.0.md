# v0.2.0 Release Notes

This release introduces major enhancements to the Middleman routing & safety layer, provides a new release readiness audit recipe, and extends CLI capabilities with connectivity pre-flight checks.

## Key Highlights

### 1. Middleman Capability Registry & Routing Validation

We added a declarative capability matrix to ensure AgentFlow only routes requests to model providers that natively support the requested capabilities:
- **Registry Matrix**: Maps provider capabilities including `streaming`, `tool-calls`, `json-response`, `smoke-test`, `token-limits`, and `timeout`.
- **Validation Gates**: Instantly rejects requests if the resolved provider lacks required capabilities, preventing wasted API calls.
- **Streaming Fallbacks**: Automatically falls back to non-streaming queries with a clear route warning on providers that do not support streaming.

### 2. Security Profiles & Custom Redaction Patterns

The Middleman's safety layer has been generalized to support configurable security profiles and project-specific secret scrubbing rules:
- **Security Profiles**: Configure behavior via `default` (redact findings), `strict` (block requests and throw `MiddlemanPolicyError`), or `off` (bypass scanner completely).
- **Custom Redactions**: Define custom regular expression patterns inside policies to redact or block proprietary tokens, credentials, or custom IDs.
- **Metadata Auditing**: Detailed routing metadata (resolved provider, model, reason, warnings, and policy profile) is logged to sprint logs (`events.jsonl`) for end-to-end transparency.

### 3. Release-Readiness Recipe

A new workflow recipe automates release sanity checks before public release:
- **`audit-changelog`**: Compares Git logs vs `CHANGELOG.md` for completeness.
- **`check-version`**: Verifies SemVer version increments against recent Git tags.
- **`validate-docs`**: Checks documentation coverage for new CLI flags or configuration options.

### 4. Smoke-Test CLI

- **Pre-flight Check**: Introduced the `pnpm ag smoke-test` command to verify model provider credentials and connectivity against gateways before initiating long-running sprints.

## Detailed Changes

- **test**: offline unit tests for custom redaction patterns and security profiles

## Upgrading

No breaking API changes are introduced. Ensure your `agentflow.config.json` is updated if you wish to leverage `customRedactions` inside your policy configurations.
