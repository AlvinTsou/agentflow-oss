Why AgentFlow?

Traditional AI coding assistants act as black-box chat interfaces. AgentFlow introduces structured state machines and automated quality gates directly in your codebase.

🔄

Quality Loop Rubrics

Every output goes through a strict multi-attempt review-and-fix loop. Code is scored against defined criteria before check-in.

📍

Git Checkpoints

The engine commits step results to a local Git workspace automatically. Rollback, review diffs, or resume seamlessly.

🛡️

Middleman Safety

An internal provider routing layer. Prevents API capability mismatch, audits route decisions, and redacts proprietary secrets.

📋

Readiness Audit

Summarizes tickets and code state into structured blocking, deferred, or nit signals before merging into production.

Built-in Recipes

Recipes define how the AI engine orchestrates steps. Explore our core workflows below.

Recipe: sdd
Mode: real-codebase
Quality Loop: Active (Score: 9+)

Spec-Driven Development (sdd) is designed for writing physical software. It discusses the brief, prototypes changes, runs your local tests, auto-patches failures in a loop, and stages the clean implementation.

Recipe: research
Mode: synthetic
Concurrency: Parallel Subagents

A thorough research engine that frames the initial query, writes a detailed research plan, runs parallel subagents to investigate details concurrently, synthesizes findings, critiques the output via an independent reviewer, and produces a final summary.

Recipe: release-readiness
Mode: real-codebase
Audit Type: Version & Docs

Automates repository checks prior to publishing. It audits the changelog for missing commits, checks package version formatting against Git tags, and cross-references source features against markdown files.
