왜 AgentFlow 인가?

기존의 AI 코딩 어시스턴트는 대개 블랙박스형 채팅 방식이었습니다. AgentFlow는 로컬 코드베이스에 구조화된 상태 기계와 자동화된 품질 관문(Quality Gate)을 도입합니다.

🔄

품질 루프 및 평가

모든 코드 산출물은 '작성-검증-수정'의 엄격한 루프를 거치며, 커밋 전 설정한 기준(Rubric)에 따라 평가 검증을 통과해야 합니다.

📍

Git 세이브포인트

엔진이 각 단계별 결과를 로컬 Git 작업 영역에 자동으로 커밋합니다. 언제든 이전 상태로 롤백하거나 작업 세션을 이어 나갈 수 있습니다.

🛡️

Middleman 라우터

LLM 성능 불일치를 방지하고, 라우팅 결정을 로깅하며, 민감 데이터를 사전에 필터링(마스킹)하는 내부 추상화 라우팅 계층입니다.

📋

Readiness 준비도 분석

최종 병합(Merge) 전 코드 및 요구사항 분석 상태를 Blocked(보류), Deferred(연기), Nit(마이너) 상태로 구분하여 리포트를 제공합니다.

내장 Recipes 워크플로우

Recipes는 AI 워크플로우 단계를 정의합니다. 핵심 동작 순서를 아래에서 확인해보세요.

Recipe: sdd
Mode: real-codebase
Quality Loop: 활성화 (목표 스코어: 9+)

Spec-Driven Development(sdd)는 실체 코드를 작성하는 스프린트에 특화되어 있습니다. 요구사항 분석, 프로토타입 작성, 로컬 테스트 실행 및 오류 발생 시 자동 수정(Quality Loop)을 거쳐 최종 반영합니다.

1. 스펙 검토
→
2. 코드 분석
→
3. 프로토타이핑
→
4. 리뷰 및 점수 검증
→
5. 자동 수정 루프
→
6. 코드 완성
Recipe: research
Mode: synthetic
Concurrency: 병렬 서브에이전트

고도화된 리서치 워크플로우. 초기 질문을 여러 하위 질문으로 세분화한 뒤 에이전트들을 병렬 가동해 세부 조사를 진행하며, 결과물 취합 후 독립 리뷰어가 이를 검증하여 보고서를 완성합니다.

1. 주제 확정
→
2. 질문 세분화
→
3. 병렬 조사 (Parallel Agents)
→
4. 정보 종합
→
5. 비판 및 검토
→
6. 보고서 완료
Recipe: release-readiness
Mode: real-codebase
Audit Type: 버전 및 문서

배포 직전 리포지토리 상태를 자동 진단합니다. 변경 이력(Changelog)에 누락된 커밋이 없는지 감사하고, package.json 버전과 Git tag의 정렬 상태 및 소스 문서와의 동기화 여부를 점검합니다.

1. 변경이력 감사
→
2. 버전에 SemVer 적용 확인
→
3. 문서 동기화 검증
→
4. readiness.json 생성

프로젝트 진행 상황

Week 6 유지보수 진행 중 상태 업데이트: 2026-06-14

공개 repository 의 OSS baseline 은 준비되어 있습니다. CI, issue templates, security policy, contribution docs, CLI reference, architecture docs, active roadmap 이 갖추어졌습니다. Core engine 은 sprint recipes 실행, artifacts 와 state 저장, quality loops, resume, readiness signals 를 지원합니다. Middleman layer 는 provider capability validation, route metadata, security profiles, 재생 가능한 route audit output 을 제공합니다. 완료된 recipe set 은 mini, research, sdd, release-readiness, pr-review, security-review 입니다.

  • OSS 기준선 완비

    Apache-2.0 라이선스로 오픈소스로 전향하였으며, 기여 가이드, 보안 정책, PR 템플릿 및 기본 CI 설정을 완료하였습니다.

  • 핵심 CLI 워크플로우

    지원 명령: init, run, resume, status, replay, approve, request-changes, force-pass, resolve.

  • 오케스트레이션 Middleman 라우팅

    기능 검증, 라우트 메타데이터 기록, 보안 프로필을 적용하는 내부 라우팅 계층을 적용하였습니다.

  • 릴리즈 준비 기능 강화

    현재 초점은 Pages accuracy, route audit examples, 차기 확장 기능입니다.

기능 세부사항

현재 사용 가능한 기능들과 향후 로드맵의 세부 내역입니다.

출시된 기능

  • CLI-first 개발 엔진

    터미널 작업에 초점을 맞춰 설계. 로컬 작업 영역 관리, Git 체크포인트를 통해 스프린트를 조율.

  • Middleman 라우팅

    LLM의 역량에 맞춰 적절한 라우팅을 수행하며, 보안 프로필과 사전 요청 검증을 처리.

  • 품질 루프 Rubrics 검증

    자동 코딩 결과물을 평가 루브릭에 맞춰 검증하고 수정 지침을 반영하여 통과할 때까지 반복 수행.

  • Recipe Workflows 및 Review Automation

    release-readiness 와 PR review recipes 를 제공하며, 구조화된 review context 를 위한 git diff ingestion 를 지원합니다.

  • Security Review Recipe

    인증, 데이터 액세스 및 안전하지 않은 로깅을 감사하고 구조화된 Verdict를 생성하는 3단계 보안 레시피입니다.

개발 계획 및 로드맵

  • 공개 문서 정렬

    multilingual Pages, recipe guides, route audit examples, maintenance logs 를 실제 구현 상태와 동기화합니다.

    • 추가 Sanitized Replay Fixtures

    public-safe route audit replay examples 를 확장해 maintainer verification 을 추적하기 쉽게 만듭니다.

    웹 대시보드 제어 화면

    CLI 기반 코어 엔진 구조가 완전히 안정화된 이후 제공할 예정인 가벼운 웹 UI 대시보드 화면.
